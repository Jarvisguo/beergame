# 啤酒分销游戏 — 玩家快速操作手册

## 游戏简介

你扮演供应链中的一个角色，与另外3名队友协作运营26周。每轮需要**收货 → 发货 → 订货**。

## 四个角色

| 角色 | 上游（向谁订货） | 下游（给谁发货） |
|------|------------------|-------------------|
| 零售商 | 批发商 | 客户 |
| 批发商 | 区域仓库 | 零售商 |
| 区域仓库 | 工厂 | 批发商 |
| 工厂 | 工厂（生产线） | 区域仓库 |

## 如何测试（单人操作）

打开**4个浏览器标签页/窗口**，分别登录不同角色。推荐4个独立窗口（非无痕窗口），因为无痕窗口之间不共享 Socket.IO 连接，测试时容易互相干扰。

## 操作步骤

### 1. 登录
- 打开游戏页面，输入一个**唯一用户名**，点击"开始"
- 系统自动分配团队和角色
- 用4个不同用户名分别登录（如：零售测试、批发测试、仓库测试、工厂测试）
- 第二次登录时，用户名输入框会自动回填上次使用的用户名

### 2. 等待组满
- 4人全部加入后，页面显示"组已满，准备开始！"
- 可以在"团队成员"面板看到每位成员的姓名、角色和牛鞭效应

### 3. 管理端开始游戏
- 打开管理端页面，输入密码，点击"开始游戏"
- 玩家端立即进入第1周

### 4. 每轮操作（每周重复）
每周按以下顺序操作：

1. **📦 接收货物** — 点击"接收货物"按钮，货物从上游到达并加入库存
2. **🚚 完成订单** — 点击"完成订单"按钮，系统自动给下游发货（扣库存、更新积压）
3. **📝 下单** — 输入订货数量（非负整数），点击"下单"提交
4. **⏳ 等待** — 等待其他3名队友都提交订单后，自动进入下一周

> **注意**：每轮提交订单后就无法再修改，所以要慎重填写订单数量。

### 5. 游戏结束
- 26周结束后游戏自动完成
- 管理端可以查看统计报表

## 界面说明

### 实时分析面板
- **成本趋势**：每周累计成本的柱状图
- **订单历史**：每周订单量的柱状图（绿色≤12 / 黄色≤16 / 红色>16）
- **牛鞭效应**：显示你当前订单的变异系数（CV），点击旁边的 **?** 按钮查看计算方法和分级标准
- **供应链状态**：正常 / 有积压 / 严重积压 / 高库存

### 团队成员表格
显示同组4人的**用户名**、**角色**、**牛鞭效应**等级，可以直观对比队友的订单波动程度：
- 绿色标签 — 稳定（CV < 10%）
- 黄色标签 — 波动（CV 10%–30%）
- 红色标签 — 剧烈（CV ≥ 30%）

## 牛鞭效应

### 什么是牛鞭效应
供应链中需求信息扭曲放大的现象——下游需求的微小变化在向上游传递时被逐级放大，导致上游供应商面临越来越大的订单波动。

### 计算方法
**变异系数 (CV) = 标准差 / 平均值**

CV 衡量订单相对于均值的离散程度。CV 越大，说明订单波动越剧烈，牛鞭效应越明显。

### 降低牛鞭效应的策略
- 信息共享：让供应链各环节了解真实需求
- 减少订货批量：避免一次性大额订货
- 稳定价格：避免促销导致的集中订货
- 避免恐慌性订货：缺货时不超额补单

## 核心指标说明

| 指标 | 含义 |
|------|------|
| 库存 | 当前持有的啤酒数量 |
| 积压 | 下游订单中暂未发出的数量 |
| 成本 | 累计持有和缺货成本（库存×0.5 + 积压×1） |
